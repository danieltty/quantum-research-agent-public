# Evidence Policy

This repository distinguishes artifact status explicitly. The goal is to show workflow shape and artifact discipline without manufacturing production history.

| Label | Meaning |
| --- | --- |
| `sample` | Hand-created or generated example showing expected artifact shape. Not a production run. |
| `demo` | Output generated by the public demo pipeline. Not a production run. |
| `sanitized_real_run` | Output derived from a real production run after manual sanitization. |
| `template` | Blank or partial structure meant to guide future artifacts. |

## Rules

- Do not manufacture run history.
- Do not imply that a demo artifact is a production artifact.
- Do not publish real artifacts unless they have been manually reviewed and sanitized.
- Every artifact should include `artifact_status`.
- Every artifact should include enough context to identify its source type.
- Sample and demo artifacts must stay visibly labeled.

## Current Artifact Set

The initial repository includes sample artifacts and a deterministic demo pipeline. It does not include sanitized production artifacts.

Future sanitized real runs should be added only after manual review and should use `artifact_status: sanitized_real_run`.
