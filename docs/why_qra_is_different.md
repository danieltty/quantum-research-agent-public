# Why QRA Is Different

QRA is not positioned as a generic chatbot or a one-shot paper summarizer. It is built around repeated research work.

## Always-On Research Loop

The private system runs a daily research loop. Each run can discover sources, analyze selected material, synthesize updates, and prepare a concise notification or digest.

## Deep Research Over One-Shot Summaries

The goal is not to produce large volumes of readable text. The goal is to identify what matters, preserve why it matters, and separate weak leads from claims worth follow-up.

## Durable Research Memory

QRA is designed to compound context over time. Later runs can use prior structured notes, active questions, relation records, and memory updates instead of starting from a blank context window.

This is different from asking a model to summarize today's papers without remembering what the system already found yesterday.

## Claim Typing

The system is designed to keep different claim types separate:

- formal
- empirical
- interpretive
- philosophical
- methodological
- review
- speculative

This matters because scientific research becomes noisy when speculative connections are presented like established results.

## Provenance-Backed Outputs

The public artifacts emphasize manifests, schemas, provenance records, and validation checks. The private system's working principle is that useful research notes should remain connected to source evidence.

## Candidate Hypotheses For Review

QRA can propose candidate hypotheses or follow-up leads. These remain explicitly human-reviewed. The point is to accelerate research judgment, not replace proof, peer review, or expert interpretation.

## Public Boundary

This repository shows the public evidence layer:

- conceptual workflow
- artifact formats
- sample and demo outputs
- sanitized operating evidence
- validation checks

It intentionally excludes private prompts, private memory, unpublished research notes, raw traces, internal scoring logic, and production code architecture.
